# triggerctl

Manage triggers via `triggerctl` (on PATH after `install.sh`). Engine docs: [triggerctl/README.md](/mnt/afs_toolcall/sunhao4/triggerctl/README.md).

> This environment often lacks bare `pip`. Use `bash install.sh` or `/opt/conda/bin/python3 -m pip install -e <repo>`.

## Mental model

- **Types from frontmatter** (no `type` field):
  - `schedule` only → time (poll)
  - `probe` only → event (poll)
  - `schedule` + `probe` → combo AND (poll)
  - `when` only (no schedule/probe) → **semantic session** (Agent self-check via hook)
- **Probe vs when**: sharp shell predicate → `--probe`; human/Agent judgment → `--when`
- **Two layers**: cheap Python detection; model execution only when DUE
- **Body** = natural-language steps for the model (not shell). Replace TODO after `add`.
- **Strict context**: only enabled semantic session triggers with `inject: true` (default) enter the hook. time/event **never** enter context. `inject: false` for statusLine/doctor guardrails.
- **Roots**: user `~/.claude/triggers/` · project `<repo>/triggers/`
- **Timezone**: `TRIGGERCTL_TZ_OFFSET=8` (default +8)
- **Hook replace (experimental, Claude only)**: `TRIGGERCTL_HOOK_REPLACE=1` in settings env
- **Hermes**: `triggerctl install --hermes` → `pre_llm_call` in `~/.hermes/config.yaml`; `triggerctl hermes-hook`
- **Codex**: `triggerctl install --codex` → `UserPromptSubmit` in `~/.codex/hooks.json`; `triggerctl codex-hook`

## Register a trigger

1. Pick root: `--root user` (global) or `--root project`
2. `triggerctl add …` (rebuilds ops index)
3. Edit the `.md` body
4. `triggerctl list` to confirm

```bash
triggerctl add nightly-report --root user --category ops --every day --at 14:30
triggerctl add on-done --root user --probe 'test -f /data/done.flag' --dedup-cmd 'stat -c %Y /data/done.flag'
triggerctl add auto-commit --root user --when 'when a feature is complete: small=commit only, large=commit and push'
```

## Other commands

```bash
triggerctl list [--root all]
triggerctl enable/disable <name>
triggerctl remove <name>
triggerctl add --from <SOURCE> [--list]
triggerctl update
triggerctl doctor
triggerctl validate [--probe-test]
triggerctl detect / poll
triggerctl sync
```

## Background poll (time/event)

```bash
triggerctl install --root user --loop --interval 60
nohup ~/.claude/triggers/run-loop.sh 60 >/dev/null 2>&1 &
```

Ask before starting long-running background processes.

## Pitfalls

- Probes must be read-only, fast, side-effect free
- Do not hand-edit `TRIGGERS.md` (ops index; regenerated by sync)
- Document safety rules in bodies (no force push, no secrets)
